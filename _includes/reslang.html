Languages

I wrote this blog post about the programming languages I had used as of 2007. All experiences detailed there are still true (except that my opinion of Java has improved). Here is an updated synopsis of my experience with less commentary about the languages themselves.

C & C++

C and C++ are very different languages, but my experiences with the two are interlocked.

I first used C At CSL in 1985, when I ported the pharmacokinetics modeling tool from Pascal to C. Five years later IESC brought in a tutor to teach the programming team C++. In the end it was decided that the C++ tools available at that time were not up to the job, but we did use object-oriented design principals despite the fact we implemented the system in C. I started using C++ for my own programs at that time.

I fist used C++ commercially for ComSat. Where wrote object-oriented C++. The at the SEC I used C++ as a type safe C, writing code that was required to compile cleaning as either C or C++.

At Epsilon and Geo-Centers I wrote everything in C++. The web service I developed for Sungard was also written in C++, using a proprietary web framework developed in house.

Python

Python is still my goto language for most tasks. It is very easy to use and has lots of good libraries and frameworks.

Java

For years I avoided Java, seeing it as a crippled dialect of C++, and only used it when required to do so (most notably by MeetingMaker & RulesPower). Since I have been using it on my own to develop Android apps I have started to appreciated it as a language in it's own right with its own strengths and weaknesses. While it is true that Java is not C++, I now see that C++ is not Java. While there are many tasks for which C++ is better suited than Java, there are others for which Java is better suited.

JavaScript

JavaScript is the rare language that the more I use it the more I like it. The only reason node.js has not replaced Python as my goto languages it that Python has good libraries that do what I need, this is less true of JavaScript.

R

I had started to play with R in 2007, but in the last five years have been using it quite a bit for several of the MOOCs I have been taking.

MatLab / Octave

I used Matlab for exploratory data analysis at Geo-Centers and G.F. Goodrich Aerospace. More recently I have used both Matlab and Octave, a GNU clone, for several MOOCs.

Scala

I had played a little with Scala before I took Coursera's Functional Programming Principles in Scala with Martin Odersky, the designer of the language. I really like it, but have not done a lot with it yet.

Lisp

I have used Lisp off an on since the early 1980's. I have used many different dialects including Scheme, Common Lisp, and Clojure but recently I have mostly been using Racket.

Ruby

I have used Ruby for several MOOCs. I like it, even though it follows Perl's, "There is more than one way to do it," philosophy, and I prefer Pythons, "there is one obvious way to do it." Unfortunately I have not found good math and science libraries for Ruby.

Prolog

I used Prolog at Ceder Systems and at Information Engineering Systems. It is not so much a language as a way of thinking about the problem.

J

While I have dabbled with J for many years, until recently I have not taken the deep dive it deserves. Now I am diving deep because I am using it on my Android.

Other Languages

Assembler was my first programming language. I have used many other languages including Pascal, PL/1, BASIC, FORTRAN, COBOL, Haskell, ML, Oz, Php, Icon, Ada, Mathmatica, Forth, and Rexx.
